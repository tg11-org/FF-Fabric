🌐 PROXMOX CLIENT API
════════════════════════════════════════════════════════════════════════════

Methods:
  GET(path, params=None, timeout=None) → dict
    • Perform GET requests to Proxmox API
    • Query parameters support
    • Automatic retry on transient failures

  POST(path, data=None, params=None, timeout=None) → dict
    • Perform POST requests to Proxmox API
    • JSON body support
    • Async task handling

  DELETE(path, params=None, timeout=None) → dict
    • Perform DELETE requests to Proxmox API
    • Async deletion support

Features:
  ✓ Token authentication (PVEAPIToken header)
  ✓ Session pooling for efficiency
  ✓ Error message extraction from Proxmox
  ✓ SSL verification control
  ✓ Configurable timeouts
  ✓ Context manager (with statement)
  ✓ Logging support

Example:
  with ProxmoxClient(base_url, token) as client:
      nodes = client.get("/api2/json/nodes")
      result = client.post("/api2/json/nodes/pve1/lxc", data={...})

🎯 KEY DESIGN PRINCIPLES
════════════════════════════════════════════════════════════════════════════

1. ABSTRACTION
   → Core/Pulse never see raw Proxmox details
   → Clean interface hides provider complexity
   → Easy to add new providers later

2. TYPE SAFETY
   → All inputs/outputs strongly typed
   → Pydantic-compatible dataclasses
   → IDE support for type hints

3. ERROR CONTEXT
   → Meaningful error messages
   → Provider error codes preserved
   → Clear debugging information

4. EXTENSIBILITY
   → Simple to add new providers (AWS, bare-metal, etc)
   → Shared models work across providers
   → ProxmoxClient is standalone/reusable

5. EFFICIENCY
   → Connection pooling via requests.Session
   → Automatic retries for reliability
   → Configurable timeouts
   → No hardcoded delays

🔄 INTEGRATION PATTERN
════════════════════════════════════════════════════════════════════════════

Core (API Layer):
  1. Create Fabric job
  2. Use type-safe models
  3. Enqueue to Pulse

  from fabric import CreateContainerRequest
  request = CreateContainerRequest(
      node_id="pve1",
      hostname="web-001",
      memory_mb=2048,
      cores=2,
      storage_gb=50
  )

Pulse (Worker Layer):
  1. Load job from queue
  2. Call Fabric provider
  3. Handle response/errors
  4. Update Core DB

  fabric = ProxmoxFabric(...)
  result = fabric.create_container(request)
  instance.provider_ref = result.provider_ref
  db.commit()

Monitoring:
  1. Query instance status
  2. Poll task progress
  3. Track completion

  status = fabric.get_instance_status(provider_ref)
  task = fabric.get_task_status(task_id)
